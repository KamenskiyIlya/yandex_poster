# Интерактивная карта мест Москвы

Данный проект представляет из себя интерактивную карту Москвы, на которой собраны интересные и необычные места для активного отдыха: начиная с музеев и выставок и заканчивая красивой дверью или урной. Проект создаётся, чтобы помочь жителям и гостям столицы найти интересные места, которые хотелось бы посетить, чтобы обратить внимание людей на места, которые обычно они не замечали и проходили мимо.

## О проекте

Данный проект написан на фреймворке Django и включает в себя:

- Сайт с интерактивной картой, на которой отображаются все места из базы данных(далее БД).
- Панель администратора - удобный интерфейс в виде сайта для взаимодействия с БД, а именно добавление записей о местах и фото, редактирование записей и их удаление.
- Команда для массовой загрузки данных о новых местах из json файлов.
- API - для получения данных о конкретном месте в формате json.
- Настроено логгирование ошибок.

## Настройка и запуск (локально)

1. У Вас должен быть установлен [python](https://www.python.org/downloads/) версии 3 и выше. Для того чтобы проверить установлен ли python, нужно ввести эту команду в терминал(cmd):
```
python --version
```
> Вывод должен быть такой, в нем будет указана версия python(если установлен): `Python 3.14.0`

2. Сохраните данный репозиторий в любое удобное место.
3. Создайте виртуальное окружение и активируйте его при помощи следующей команды:
```
python -m venv venv
venv\Scripts\activate # на Windows
source venv/bin/activate # на Linux
```
4. Установите необходимые зависимости введя следующую команду:
```
pip install -r requirements.txt
```
5. Создайте в главной папке проекта файл для данных переменного окружения - `.env`. Сразу запишите в него секретный ключ для проекта Django(о нем никто не должен знать) в следующем формате:
```
SECRET_KEY='Ваш_секретный_ключ'
```
6. Примените миграцию к БД при помощи команды:
```
python manage.py migrate
```
7. Создайте суперпользователя для доступа к панели админа. Введите команду, а после укажите login, password, mail:
```
python manage.py createsuperuser
```
8. Запустите локальный сервер. На данном этапе он пригоден только для ознакомления и разработки, для продакшена он не подходит. Если увидели следующее сообщение в командной строке - значит все удачно запустилось:

<img width="1086" height="254" alt="image" src="https://github.com/user-attachments/assets/53d7017d-bf07-4c3a-bb45-bdee84f46114" />

После этого локальный сайт будет доступен по ссылкам:  http://127.0.0.1:8000/ или http://localhost:8000/. В панель администратора можно попасть по ссылке http://127.0.0.1:8000/admin/ , при входе указываете логин и пароль, которые задавали при создании superuser на этапе 7. Работает API сайта и выдает json с информацией о месте при переходе по ссылке следующего формата - http://localhost:8000/places/2/ , где 2 это id места в БД. Если id с указанным номером не существует API выдаст ошибку 404.

## Загрузка данных из JSON файлов

Проект включает в себя удобную команду для массовой загрузки данных о местах и json файлов. Это полезно, когда нужно добавить информацию о множестве мест, чтобы не создавать по одной записи в БД руками. Команда ожидает получить следующую структуру json файлов:

```json
{
    "title": название места,
    "imgs": [ # Источники фото
        "https://raw.githubusercontent.com/devmanorg/where-to-go-places/master/media/1f09226ae0edf23d20708b4fcc498ffd.jpg",
        "https://raw.githubusercontent.com/devmanorg/where-to-go-places/master/media/6e1c15fd7723e04e73985486c441e061.jpg",
        "https://raw.githubusercontent.com/devmanorg/where-to-go-places/master/media/be067a44fb19342c562e9ffd815c4215.jpg",
    ],
    "description_short": короткое описание,
    "description_long": длинное описание,
    "coordinates": {
        "lng": долгота,
        "lat": широта,
    }
}
```
> Название места - title должно быть уникальным, иначе команда не создаст новое место, т.к. такое место уже существует в БД.

### Как пользоваться

- Подготовьте папку с json файлами в любом удобном месте
- Запустите встроенную команду введя в терминале:

```
python manage.py load_place путь_до_вашей_папки
```
При выполнении команды происходит следующее:

1. Находятся все json файлы в указанной директории
2. Для каждого файла:
- Достается информация о месте
- На основе этой информации создается новое место в БД
- Скачиваются все фото из поля `imgs`
- Фото привязываются к месту с правильным порядком
- Сами фото сохраняются по адресу `media/place_images/`
3. В терменале выводится статистика по результату загрузки:

## API

Если перейти во ссылке: http://localhost:8000/places/<id_нужного_места>/, сайт вернет ответ со всей информацией о месте с указанным id. Структура ответа следующая:

```json
{
  "title": название места,
  "imgs": [
    источники фото,
  ],
  "description_short": короткое описание,
  "description_long": длинное описание,
  "coordinates": {      # координаты
    "lat": широта,
    "lng": долгота,
  }
}
```

## Логгирование

В проекте настроено базовое логгирование ошибок и событий. При запуске проекта создается папка `logs/`, в которую сохраняются логи, также логи выводятся в командную строку. Если Вас не достаточно стандартно настроенного логгирования, Вы можете перенастроить логгер под свои нужды в соответствии с [оф. документацией django](https://docs.djangoproject.com/en/6.0/topics/logging/).

- Название файлов - `log_file.log`
- Ротация - файл автоматически сохраняется при достижении веса в 5Mb, последующие логи записываются в новый файл(всего хранится не больше 3х файлов)
- Уровни - логгер фиксирует сообщения начиная с уровня `INFO`

## Безопасность

Если Вы намерены запускать сайт в боевом режиме на сервере и пускать его в продакшн, предусмотрите вопросы безопасности.
Обязательно в переменном окружении настройте все указанные параметры, они по стандарту выставлены для возможности запустить проект локально. Без них запуск на сервере не безопасен. Откройте файл `.env` и определите следующие параметры:
```
DEBUG=False
ALLOWED_HOSTS=Ваши_домены_через_запятую
CSRF_COOKIE_SECURE=TRUE
SESSION_COOKIE_SECURE=TRUE
SECURE_SSL_REDIRECT=TRUE
SECURE_HSTS_SECONDS=60
SECURE_HSTS_INCLUDE_SUBDOMAINS=TRUE
SECURE_HSTS_PRELOAD=TRUE
```
- DEBUG - режим отладки, при разработке нужен, но в продакшене пользователю не должна выводится отладочная информация в случае ошибки.
- ALLOWED_HOSTS - домены, на которых работает сайт(указывать через запятую).
- CSRF_COOKIE_SECURE - django проверяет, что cookie отправляются только через защищённое (HTTPS) соединение.
- SESSION_COOKIE_SECURE - сессионные куки будут передавать только по HTTPS.
- SECURE_SSL_REDIRECT - Принудительное перенаправление на HTTPS, на сайт не попасть через не защищенный запрос HTTP.
- SECURE_HSTS_SECONDS - настройка в Django, которая управляет использованием HTTP Strict Transport Security (HSTS). Она определяет длительность (в секундах), в течение которой браузер будет принудительно использовать HTTPS для доступа к сайту.
- SECURE_HSTS_INCLUDE_SUBDOMAINS - применяет HSTS ко всем поддоменам.
- SECURE_HSTS_PRELOAD - при значении True Django добавляет директиву preload в заголовок HSTS.

⚠️Перед выставлением этих настроек убедитесь, что понимаете, что каждая из них делает. Выставление некоторых настроек может привести к необратимым изменениям сайта.

Дополнительно проверьте себя нижеуказанной командой, django даст рекомендации по безопасности проекта:
```
python manage.py check --deploy
```
Также перед запуском сайта в боевом режиме, советую ознакомиться с рекомендациями по настрокам безопасности в [оф. документации django](https://docs.djangoproject.com/en/5.2/howto/deployment/checklist/).
